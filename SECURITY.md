# Security Policy

## Supported versions

Security fixes are applied to the **current `main` branch** only. Older tags or branches are not supported unless explicitly stated in a release note.

| Version / branch | Supported |
|------------------|-----------|
| `main` (latest)  | Yes       |
| Other branches   | No        |

## Reporting a vulnerability

If you discover a security vulnerability, **please do not open a public GitHub issue**.

Instead, report it privately by email:

**yasinjir@outlook.com**

Include as much detail as you can:

- Description of the issue and potential impact
- Steps to reproduce
- Affected endpoints or components (if known)
- Any suggested fix or mitigation

We will acknowledge your report as soon as possible and work on a fix. We appreciate responsible disclosure.

## What not to report publicly

- Database credentials, API keys, or `.env` contents found in commits (contact us privately; rotate secrets immediately)
- Issues that require access to production Vercel or Supabase dashboards without a reproducible application-level flaw

## Safe harbor

We support good-faith security research that follows this policy and does not harm users or production data.
